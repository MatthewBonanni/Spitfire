Explicit time integration with user-defined methods¶

Highlights

  • solving ordinary differential equations with your own methods
    • custom eighth-order Runge-Kutta method
    • coding up a second-order Adams-Bashforth method for use in Spitfire

Introduction¶

In previous demonstrations you've seen Governor and ForwardEuler and other Spitfire classes used to solve some simple ordinary differential equations. Spitfire's abstraction of time integration techniques makes it possible for you to "plug in" methods of your own, which is the focal point of this notebook.

Two ways of building your own time-stepping methods are covered:

  1. Using the GeneralAdaptiveExplicitRungeKutta class in spitfire.time.methods to provide Butcher coefficients for general, explicit Runge-Kutta methods (possibly with error-based adaptive time-stepping)
  2. Writing your own Method classes for a Governor instance

We will use the first method to build two different Runge-Kutta methods, and we will show how the second method can actually enable the use of Adams-Bashforth methods not provided by Spitfire.

Custom Runge-Kutta methods with GeneralAdaptiveExplicitRungeKutta¶

This class allows you to define the Butcher tableau for any explicit Runge-Kutta method. Below we first build the Forward Euler method and then a more interesting example - the two-stage, second-order method of Ralston.

import numpy as np

from spitfire import GeneralAdaptiveExplicitRungeKutta as GERK

                                            A=np.array([[0.]]),
                                            b=np.array([1.]),
                                            order=1)
runge_kutta_methods['Ralston'] = GERK(name='Ralston',
                                      A=np.array([[0., 0.],
                                               [2. / 3., 0.]]),
                                      b=np.array([0.25, 0.75]),
                                      order=2)

Now we can go ahead and solve the exponential decay problem with each method, following code from earlier demonstrations. Note how accurate the eighth-order method is compared to Forward Euler and the Ralston method.

import matplotlib.pyplot as plt

governor = Governor()
governor.do_logging = False
governor.termination_criteria = FinalTime(1.)

dt = 0.075
k = -10.
y0 = np.array([1.])
rhs = lambda t, y: k * y

data = SaveAllDataToList(initial_solution=y0)
governor.custom_post_process_step = data.save_data

%matplotlib notebook
for name, method in runge_kutta_methods.items():
    data.reset_data(initial_solution=y0, initial_time=0.)
    governor.integrate(right_hand_side=rhs,
                       initial_condition=y0,
                       controller=dt,
                       method=method)
    plt.plot(data.t_list, data.solution_list, '-', label=name)

plt.plot(data.t_list, y0 * np.exp(k * data.t_list), 'o', markerfacecolor='w', label='exact')

plt.xlabel('t')
plt.ylabel('y')
plt.legend(loc='best')
plt.grid()
plt.show()

Adams-Bashforth method with a custom class¶

Now we show how to write a new method that can be used with a Governor instace. Spitfire doesn't provide linear multistep methods, but here we can pretty quickly code up the second-order Adams-Bashforth method. Our class derives from the Spitfire's TimeStepper class, to which it provides its name and order of accuracy in the super().__init__ call. Next, the single_step method is defined and written so that it returns a Spitfire StepOutput object. And that's it - we just have to make sure that the update is computed correctly in the single_step method.

The second-order Adams-Bashforth method is a two-step method that uses the right-hand side at both the current time level and the prior one. The next time level, with solution $y^{n+1}$, is computed with

$$ y^{n+1} = y^{n} + \frac{\Delta t}{2}\left(3 r(t^n,y^n) - r(t^{n-1},y^{n-1})\right) $$

where $r(t,y)$ is the right-hand side function of the differential equation. This update can't be performed on the first time step and so we simply use the Forward Euler update $y^{n+1} = y^{n} + \Delta t r(t^n, y^n)$. This is not strictly accurate to second-order, but it works well enough in practice.

A key detail in implementing linear multistep methods is the challenge of handling a variable time step history. We simply ignore this difficulty here, so the class below will only be correct for a fixed time step.

Note that custom time-stepper methods like this can return quite a bit more information, such as error estimates for time step adaptativity, in the StepOutput class, but that's a bit more advanced and we suggest interested developers check out the source code in the spitfire.time.methods.py file.

In [5]:
from spitfire.time.methods import StepOutput, TimeStepper

class AdamsBashforth2(TimeStepper):
    def __init__(self):
        super().__init__(name='Adams-Bashforth-2', order=2)
        self._rhs_nm1 = None

    def single_step(self, state, t, dt, rhs, *args, **kwargs):
        rhs_n = rhs(t, state)
        
        if self._rhs_nm1 is None:
            update = dt * rhs_n   # if first step, use Forward Euler
        else:
            update = 0.5 * dt * (3. * rhs_n - self._rhs_nm1)
            
        self._rhs_nm1 = np.copy(rhs_n)    
        return StepOutput(solution_update=update)

Now this class may simply be provided to the integrate method of a Governor instance just like the other explicit methods considered so far.

Verification¶

To make sure that we coded our new Adams-Bashforth method correctly, we're going to verify by establishing its order of accuracy. This consists of running simulations at different time step sizes and observing how the error (against an exact solution) decreases. A first-order method will see a 10x decrease in error for a 10x decrease in the step size, while a second-order method would see a 100x decrease and a fourth order method would see a 10000x decrease.

To simplify this we define the compute_error method below that takes in a time-stepper method and a step size and returns the error in solving the exponential decay problem to a specified time.

In [7]:
def compute_error(method, time_step_size):
    local_governor = Governor()
    local_governor.do_logging = False
    local_governor.termination_criteria = FinalTime(0.5)
    _, yf, tf, _ = local_governor.integrate(right_hand_side=lambda t, y: k * y,
                                            initial_condition=y0,
                                            controller=time_step_size,
                                            method=method)
    return np.abs(yf - (y0 * np.exp(k * tf)))[0]

Now we import a few more methods provided by Spitfire for comparison's sake and run compute_error for each method across a range of time step sizes. This uses a Python list comprehension to simplify the construction of the list of errors, and simply saves off the results in a dictionary. This sort of pattern is really convenient in many uses of Spitfire.

Following this we use NumPy's polyfit function to obtain the slopes of the error over time step size results (in logarithmic scale). This shows that Spitfire's methods and our new second-order Adams-Bashforth method are all solving the problem to their design order of accuracy.

In [8]:
from spitfire import ForwardEuler, \
    ExplicitRungeKutta4Classical, \
    ExplicitRungeKutta2Ralston, \
    ExplicitRungeKutta3Kutta

dt_list = [1e-4, 2e-4, 5e-4, 1e-3, 2e-3, 5e-3, 1e-2]

error_dict = dict()
error_dict['FE'] = [compute_error(ForwardEuler(), dt) for dt in dt_list]
error_dict['RK2'] = [compute_error(ExplicitRungeKutta2Ralston(), dt) for dt in dt_list]
error_dict['RK3'] = [compute_error(ExplicitRungeKutta3Kutta(), dt) for dt in dt_list]
error_dict['RK4'] = [compute_error(ExplicitRungeKutta4Classical(), dt) for dt in dt_list]
error_dict['AB2'] = [compute_error(AdamsBashforth2(), dt) for dt in dt_list]

order_dict = dict()
for method in error_dict:
    order_dict[method] = np.polyfit(np.log(dt_list[2:]), np.log(error_dict[method][2:]), 1)[0]

%matplotlib notebook
for method in error_dict:
    plt.loglog(dt_list, error_dict[method], 'o--', label=f'{method}, order={order_dict[method]:.2f}')
plt.legend(bbox_to_anchor=(1.04,0.5), loc="center left", borderaxespad=0)
plt.xlabel('time step size')
plt.ylabel('global error')
plt.grid()
plt.tight_layout()
plt.show()

Conclusions¶

In this example we've shown how to employ user-defined explicit Runge-Kutta methods and custom classes, in this a second-order Adams-Bashforth method, to solve a simple ordinary differential equation. Verifying the order of accuracy of the methods suggests confidence that they are implemented correctly.
